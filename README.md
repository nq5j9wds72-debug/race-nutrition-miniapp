# Race Nutrition Mini App

Telegram Mini App для быстрого расчёта плана питания на гонку: углеводы, жидкость, натрий и ориентир по гелям.

---

## Что это за проект

Проект помогает спортсмену быстро получить понятный план питания на гонку прямо внутри Telegram Mini App.

Текущая версия умеет:

- открываться внутри Telegram как Mini App
- проверять Telegram initData на сервере
- считать план по основным полям
- учитывать дополнительные поля, если пользователь хочет уточнить расчёт
- сохранять историю расчётов пользователя
- показывать историю расчётов во frontend
- показывать разные состояния результата:
  - пустое состояние
  - состояние загрузки
  - состояние ошибки
  - готовый результат

## Что уже реализовано

### Telegram Mini App

Реализовано:

- запуск внутри Telegram WebApp
- вызовы `tg.ready()` и `tg.expand()`
- получение `initData`
- использование Telegram-пользователя в Mini App

### Авторизация

Реализовано:

- отправка `initData` на backend
- серверная проверка Telegram initData
- использование подтверждённого пользователя для защищённых endpoint

### Расчёт питания

Реализовано:

- расчёт по основным полям:
  - тип гонки
  - длительность
  - вес
  - температура
  - формат питания
  - переносимость углеводов

- поддержка дополнительных полей:
  - интенсивность нагрузки
  - влажность
  - дистанция
  - потоотделение
  - набор высоты
  - профиль потерь натрия

### Экран результата

Реализовано:

- пустое состояние:
  - `План пока не рассчитан`
- loading-state:
  - `Считаю план`
- error-state:
  - `Расчёт не выполнен`
  - `Не удалось получить результат`
- готовый экран результата:
  - заголовок `Твой план на гонку`
  - ключевые KPI:
    - углеводы
    - жидкость
    - натрий
  - длительность
  - короткое пояснение
  - ключевая мысль
  - детальные карточки:
    - Углеводы
    - Жидкость
    - Натрий
    - Эквивалент в гелях
    - Как действовать на гонке
    - Что учесть

### История расчётов

Реализовано:

- backend endpoint `/api/history`
- возврат только расчётов текущего подтверждённого пользователя
- загрузка истории во frontend
- отображение пустой истории
- отображение ошибки загрузки истории
- обновление истории после нового расчёта
- новые записи попадают наверх списка

### UI-улучшения Day 32–34

Сделано:

- убрана верхняя debug-плашка из видимого интерфейса
- убрана нижняя техническая строка auth/calc из видимого интерфейса
- дополнительные поля вынесены в отдельный раскрываемый блок
- логика управления дополнительными полями упрощена:
  - одна кнопка:
    - `Показать дополнительные поля`
    - `Скрыть дополнительные поля`
- убрано дублирование управления открытием/закрытием
- кнопка `Рассчитать план` перенесена вниз формы:
  - в закрытом состоянии находится после основного ввода
  - в открытом состоянии находится после дополнительных полей
- интерфейс стал более showable для демонстрации MVP
