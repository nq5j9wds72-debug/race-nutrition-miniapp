# Race Nutrition Mini App

Telegram Mini App для быстрого расчёта плана питания на гонку: углеводы, жидкость, натрий и ориентир по гелям.

---

## Что это за проект

Проект помогает спортсмену быстро получить понятный план питания на гонку прямо внутри Telegram Mini App.

Текущая версия умеет:

- открываться внутри Telegram как Mini App
- проверять Telegram initData на сервере
- считать план по основным полям
- учитывать дополнительные поля, если пользователь хочет уточнить расчёт
- сохранять историю расчётов пользователя
- показывать историю расчётов во frontend
- показывать разные состояния результата:
  - пустое состояние
  - состояние загрузки
  - состояние ошибки
  - готовый результат

## Backend API

### `POST /api/auth`

Назначение:

- проверить Telegram initData
- подтвердить пользователя
- создать или обновить пользователя в БД после успешной проверки

Ответ при успехе:

- `ok`
- `user`
- `authDate`

### `POST /api/calc`

Назначение:

- выполнить основной расчёт плана питания
- использовать только подтверждённого Telegram-пользователя
- сохранить успешный расчёт в историю пользователя

Ответ:

- `ok`
- `errors`
- `warnings`
- `normalized_input`
- `result`
- `plan`

### `GET /api/history`

Назначение:

- вернуть последние расчёты текущего подтверждённого пользователя

Ответ:

- короткий список последних расчётов
- только для текущего пользователя
- без доступа к чужим данным

### `GET /health`

Назначение:

- проверить, что backend работает

## Хранение данных

Используется Postgres.

Текущая продуктовая память MVP хранит:

- пользователей
- историю успешных расчётов

### Таблица `users`

Хранит подтверждённых пользователей после успешного Telegram auth.

### Таблица `calculations`

Хранит успешные расчёты пользователя:

- входные параметры
- formula version
- результат
- warnings
- время расчёта

---

## Безопасность

Реализовано:

- серверная проверка Telegram initData
- protected endpoints для расчёта и истории
- пользователь получает только свои расчёты
- данные Telegram с клиента не используются как доверенный источник без серверной проверки

Важно:

- `initDataUnsafe` можно использовать только для UI, но не как основание для доверенного доступа
- BOT token должен храниться только на сервере в переменных окружения
